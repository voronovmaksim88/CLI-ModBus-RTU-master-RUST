# RUST-CLI-ModBus-Master

## 📝 Описание

Консольное приложение на языке Rust для работы с устройствами по протоколу **Modbus RTU** через интерфейс **RS-485**. Программа предоставляет удобный интерактивный интерфейс для настройки подключения, управления регистрами и циклического опроса устройств.

## ✨ Основные возможности

### Управление подключением
- 🔍 Автоматическое сканирование доступных COM-портов
- ⚙️ Интерактивная настройка параметров связи:
  - Выбор COM-порта
  - Адрес устройства Modbus (1-240)
  - Скорость передачи данных (2400-115200 бод)
  - Четность (None/Even/Odd)
  - Количество стоп-битов (1/2)
- 💾 Сохранение и загрузка настроек из JSON файла

### Управление регистрами
- 📋 Просмотр списка регистров с подробной информацией
- ➕ Добавление новых регистров
- ❌ Удаление регистров
- 🔄 Сортировка регистров по адресу
- ✏️ Изменение параметров регистров (в разработке)

### Опрос устройств
- 🔄 Циклический опрос активных регистров (интервал 1 секунда)
- ⏱️ Обработка таймаутов запросов
- 📊 Отображение данных в реальном времени с цветовым кодированием
- 🚨 Счетчик ошибок связи
- 🎨 Цветной вывод для удобства восприятия

## 🔧 Поддерживаемые типы данных

### Типы переменных
- `bool` - булево значение (1 регистр)
- `u16` - беззнаковое 16-битное целое (1 регистр)
- `i16` - знаковое 16-битное целое (1 регистр)
- `u32` - беззнаковое 32-битное целое (2 регистра)
- `i32` - знаковое 32-битное целое (2 регистра)
- `float` - число с плавающей точкой IEEE 754 (2 регистра)

### Типы регистров Modbus
- `input_register` - входной регистр (функция чтения 0x04)
- `holding_register` - регистр хранения (функция чтения 0x03)
- `coil` - дискретный выход (функция чтения 0x01)
- `discrete_input` - дискретный вход (функция чтения 0x02)

## 📋 Требования

- **Rust** версии 2024 edition или выше
- **Windows** (с поддержкой ANSI цветового вывода)
- Доступный **COM-порт** для подключения к устройству RS-485

## 🚀 Установка

1. Клонируйте репозиторий:
```bash
git clone <URL репозитория>
cd rust-cli-modbus-master
```

2. Соберите проект:
```bash
cargo build --release
```

3. Исполняемый файл будет находиться в `target/release/rust-cli-modbus-master.exe`

## 📖 Использование

### Запуск программы

**В режиме разработки:**
```bash
cargo run
```

**В режиме release:**
```bash
./target/release/rust-cli-modbus-master.exe
```

### Главное меню

После запуска программы вы увидите главное меню:

```
=== Modbus RTU Server ===

Выберите действие:
  1 - Показать настройки связи
  2 - Изменить настройки связи
  3 - Начать опрос
  4 - Регистры
  9 - Выйти
```

### Первоначальная настройка

**При первом запуске** программа автоматически создаст файл `connect_settings.json` с настройками по умолчанию:
- COM-порт: COM1
- Адрес устройства: 1
- Скорость: 9600 бод
- Четность: None
- Стоп-биты: 1

**Для изменения настроек:**
1. Выберите пункт **2** (Изменить настройки связи)
2. Следуйте интерактивным подсказкам:
   - Выберите COM-порт из списка доступных
   - Укажите адрес устройства (1-240)
   - Выберите скорость передачи данных
   - Выберите четность
   - Выберите количество стоп-битов
3. Настройки будут сохранены в файл `connect_settings.json`

### Настройка регистров

1. Выберите пункт **4** (Регистры) в главном меню
2. Доступные действия:
   - **1** - Показать регистры
   - **2** - Удалить регистр
   - **3** - Добавить регистр
   - **4** - Изменить регистр
   - **5** - Отсортировать по адресу
   - **9** - Назад в главное меню

### Начало опроса

1. Убедитесь, что настройки связи заданы (пункт 2)
2. Убедитесь, что в файле `tags.csv` есть активные регистры
3. Выберите пункт **3** (Начать опрос)
4. Программа начнет циклический опрос устройства
5. Для остановки нажмите **Ctrl+C**

## ⚙️ Конфигурационные файлы

### connect_settings.json

Файл с настройками подключения:

```json
{
  "connection": {
    "port": "COM4",
    "device_address": 1,
    "baud_rate": 115200,
    "parity": "None",
    "stop_bits": 1
  },
  "metadata": {
    "last_updated": "2025-08-13T09:00:04.176600100+00:00",
    "version": "1.0",
    "description": "Настройки подключения для Modbus RTU через RS-485"
  }
}
```

### tags.csv

Файл с конфигурацией регистров (разделитель - точка с запятой):

```csv
name;description;address;var_type;modbus_type;enabled
Konf_EK;Наличие электрокалорифера (ЭК);0;bool;coil;true
Konf_YZY;Наличие ультразвукового увлажнителя (УЗУ);1;bool;coil;true
rezhim1;Режим установки, 0-стоп, 1-пуск.;10;u16;input_register;true
AlarmWord1;1-ое аварийное слово пользовательское;20;u16;input_register;true
AlarmWord2;2-ое аварийное слово системное;21;u16;input_register;true
sec_count;счётчик секунд после старта программы;10002;u32;input_register;true
Proc_YZY;Мощность УЗУ, %;150;float;input_register;true
```

**Поля CSV:**
- `name` - имя регистра (уникальный идентификатор)
- `description` - описание регистра
- `address` - адрес регистра в Modbus (0-65535)
- `var_type` - тип переменной (bool/u16/i16/u32/i32/float)
- `modbus_type` - тип регистра Modbus (input_register/holding_register/coil/discrete_input)
- `enabled` - флаг активности (true/false)

## ⚠️ Устранение неполадок

### Программа не видит COM-порты
- Убедитесь, что драйверы USB-UART установлены
- Проверьте Диспетчер устройств Windows
- Запустите программу от имени администратора

### Ошибка подключения к устройству
- Проверьте правильность настроек (скорость, четность, стоп-биты)
- Убедитесь, что адрес устройства указан правильно
- Проверьте физическое подключение RS-485

### Таймауты при опросе
- Увеличьте таймаут в коде (по умолчанию 1000 мс)
- Проверьте качество соединения
- Уменьшите количество опрашиваемых регистров
